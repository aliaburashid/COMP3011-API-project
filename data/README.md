# FlickGallery — Data & Seed

This folder contains all data sources and configuration used to seed the FlickGallery database (authors, posts, comments). All external datasets are cited for academic use.

---

## Overview

| Purpose | Source | File(s) | Optional? |
|---------|--------|---------|-----------|
| **Author profiles** | Top Instagram Influencers (Kaggle) | `instagram-influencers.csv` | No — or use `seed-authors.json` |
| **Profile & post images** | Celebrity Database (Kaggle) | `profile-images.js`, `post-images.js` | Yes — Unsplash used as fallback |
| **Fallback authors** | Built-in | `seed-authors.json` | Used when no CSV present |

---

## Quick start

### Minimum setup (no external downloads)

```bash
npm run seed
```

Uses `seed-authors.json` and [Unsplash](https://unsplash.com/) images. No downloads required.

---

### Full setup (Kaggle datasets)

**1. Author data — Top Instagram Influencers**

- Download: [Top Instagram Influencers Dataset (Cleaned)](https://www.kaggle.com/datasets/surajjha101/top-instagram-influencers-data-cleaned)
- Place the CSV here and name it: **`instagram-influencers.csv`**
- Run `npm run seed`

**2. Profile & post images (optional) — Celebrity Database**

- Download: [Celebrity Database (332 names)](https://www.kaggle.com/datasets/mhmaslam/celebrity-database-332-names)
- Extract into: **`data/dataset/`**
- Run: `npm run filter-and-import-kaggle`
- Run: `npm run seed`

The import script matches influencers to celebrity folders, copies images to `public/images/`, and generates `profile-images.js` and `post-images.js`. Authors without matched images (e.g. `nba`) use Unsplash URLs instead.

---

## CSV column mapping

The seed script maps CSV columns (case-insensitive) to author fields:

| Author field | CSV column(s) | Fallback |
|--------------|---------------|----------|
| `name` | `Name`, `Username`, `channel_info` | `Influencer N` |
| `email` | — | `username@example.com` |
| `password` | — | `seedpass1` (hashed) |
| `bio` | `Category`, `Followers`, `Country` | Built from available fields |
| `location` | `Country` | — |
